# Cursivis Demo Scenarios

## Scenario 1: Smart Context Response

1. User selects text in a browser, document, or editor
2. User presses `Trigger`
3. Cursivis decides the best action automatically
4. Result appears in the orb/result UI

Success criteria:

- the action matches the selected context
- the result feels useful without manual prompting

Example selections:

- French paragraph -> translation
- article -> summary
- source code -> explanation
- broken code -> debugging guidance

## Scenario 2: Guided Orb Actions

1. User selects text
2. User switches to Guided mode
3. User presses `Trigger`
4. Relevant options appear around the orb
5. Dynamic options expand after a short delay
6. Custom remains available

## Scenario 3: Talk + Selection

1. User selects text
2. User holds `Talk`
3. User says something like `count the number of words in this`
4. Cursivis combines the selection with the spoken instruction
5. Result is generated from both

## Scenario 4: Snip-it Image Flow

1. User triggers image capture
2. User draws a region
3. Cursivis analyzes that exact region
4. Result appears in the orb/result UI

## Scenario 5: Browser Action

1. User selects a live task such as MCQs, a form, or an email
2. User runs `Trigger`
3. Cursivis generates the useful result
4. User presses `Take Action`
5. The browser task is executed in the current workflow

Example outcomes:

- MCQs auto-filled
- email reply drafted in the compose surface
- form fields completed in the live browser tab

## Scenario 6: Logitech Hardware Story

1. User navigates Cursivis through MX Creative Console / Actions Ring style controls
2. Trigger, Talk, Snip-it, and Action become physical workflow controls
3. The hardware feels like an intent layer for the active app

This is the key product narrative for Logitech.
